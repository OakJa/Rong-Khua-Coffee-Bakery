1. useReducer แทน useState ในตะกร้าสินค้า

ตะกร้ามี action หลายแบบที่สัมพันธ์กัน (เพิ่ม/ลบ/เพิ่มจำนวน/ลดจำนวน/ล้าง) ถ้าใช้ `useState` เดี่ยวๆ ต้องเขียน `map`/`filter`
 กระจายหลายจุด เสี่ยง bug และดูแลยาก `useReducer` รวม logic ทั้งหมดไว้ที่ `cartReducer` จุดเดียว component แค่ `dispatch({ type })` 
 ทำให้ตามรอย เทส และขยาย action ใหม่ง่ายกว่า

2. useMemo ช่วยเพิ่มประสิทธิภาพอย่างไร

ใช้กรองเมนู (`filteredItems`) และคำนวณยอดรวม/จำนวนชิ้น (`total`, `count`) — ถ้าไม่ใช้ ทุกครั้งที่ re-render
 (เช่น เปิด/ปิดตะกร้า) จะคำนวณใหม่ทั้งที่ข้อมูลต้นทางไม่เปลี่ยน `useMemo` จำผลลัพธ์ไว้ คำนวณใหม่เฉพาะตอน dependency
  เปลี่ยนจริง ลดงานคำนวณซ้ำซ้อน

3. useCallback มีประโยชน์อย่างไร

ใช้ตอนส่งฟังก์ชัน (เช่น `addItem`, `removeItem`) ให้ child ที่ห่อด้วย `React.memo` (`MenuItem`, `CartItem`)
 เพื่อคง reference ฟังก์ชันไม่ให้เปลี่ยนทุก render **ถ้าไม่ใช้:** ฟังก์ชันใหม่ถูกสร้างทุกครั้งที่ parent re-render ทำให้
  `React.memo` เข้าใจผิดว่า prop เปลี่ยน แล้ว re-render การ์ดทุกใบโดยไม่จำเป็น — ระบบช้าลงเมื่อรายการมีจำนวนมาก

4. useContext เทียบกับ Props Drilling

Props Drilling คือส่งค่าผ่าน props ทีละชั้นแม้ component กลางทางไม่ได้ใช้ ทำให้โค้ดรกและแก้ยาก ในโปรเจกต์นี้ถ้าไม่ใช้
 context ต้องส่ง `theme` ผ่าน `App → Header → ThemeToggle` ทั้งที่ `Header` ไม่เกี่ยว `useContext` ให้ component
  ดึงค่าตรงจาก Provider ได้เลยไม่ว่าจะอยู่ลึกแค่ไหน เหมาะกับข้อมูล global เช่น ธีม แต่ข้อเสียคือค่าที่เปลี่ยนจะทำให้ทุก 
  component ที่ใช้ context นั้น re-render พร้อมกันหมด
